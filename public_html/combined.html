This is my website. I have implemented many new features. For my website I focused on Bootstrap Components, Express NodeJS, JavaScript, and Dynamic Content. These are explained below


Bootstrap Components

  • Jumbotron
  • Carousel
  • Cards
  • Callout Info
  • NavBar with Search Bar
  • Breadcrumb
  • Various Forms

The site may contain more Bootstrap component then I have listed here. All of the content in the webpage is inside a Jumbotron except for the NavBar and footer. This keeps a consistent layout and style. There is an image carousel on the home page, which makes the home page stand out. You can press the arrows to change the image or wait for it to change. I use cards to display all of the Pet products on the pet toys and pet food page. I also use cards for the customer feedback tables on both of those pages. This is a way of making the products and the customer feedback table look pleasant.

On the home page I have a callout info box we asks people if they have signed up to the website. This is a way to draw attention to the viewer, it uses CSS to look more appealing. The Navigation uses Bootstrap's NavBar it includes a search bar. I use breadcrumb which is the first code inside the jumbotron just above the logo. Breadcrumb indicates where the current page's location is in a website. It shows a nvaigational heirachy. It needs to be structured to avoid confusion. Breadcrumb is implemented using a list with a breadcrumb class. The item higher up the list is higher in the heirachy. The number of forms throughtout this webpage all us Bootstrap to implement them. This helps make the forms look professional and makes sure they work correctly.


JavaScript & Dynamic Content

  • Form Validations
  • Dynamic Greeting
  • Mouse Hover
  • Add Feedback
  • External Google Maps
  • Date Time Functions
  • Other functions

The login page, signup page, and contact page all have JS form validation features. The features they have are like what we did in previous practicals, however I have added some new features. On the signup page there is a password field and confirm password field. These two fields must match for the confirm password field to to be valid. I created this by using if (inputpassword) == (input_repeatpassword) green matches! else doesnt match. I think this is a feature that makes it easier to create a password properly.


I added a dynamic greeting like what we did in the practical, when someone clicks the button "Display Greeting" on the home page it will display a message. When you hover over the message with your mouse, the background will become orange. When you move the mouse off the message it will return to normal. I made this using the mouseHilight and mouseNot functions. The dynamic greeting involved the use of making greetings times, getting dates, and using switch values.


The feedback tables on the pet food and pet toys pages both use JS to update the card and add feedback. It's like what we did in the workshops but I have added another feature where you must select a product along with a rating. When a user adds feedback it will display their name, feedback, rating, and comment. As well as placing their input into the positive experiecnce table or negative experience table.


I added an externally hosted Google Maps on my contact web page. This is made with JS and is a dynamic external component to my webpage. I used a function initMap. Then I found the latitude and longtitude for Geelong so a marker is placed on the map. I made an API key on the google website and used that for this. It is also styled with CSS. The map doesn't work fully properly because I think you may have to pay or something.


For my order POST page I created a function that will get a date then plus 7 days for the devlivery time. I called the function addDays. So I can now get the orderDate and add 7 days. This is a good example of a JS Date function. I probably have other functions throughout the website. I hope I haven't forgot!

Feedback Table

The feedback table I implemented is simillar to what we did in the practicals. However, I added the ability to select which product you bought which is then added to the table. I researched this just by using the course material and my own understanding. It seemed a simple and neat addition to implement. It just required creating a new input option and then adding a new document element to the JavaScript code. If you want to implement this feature check out the code below

    
      var txt

      const badFeedback = [];
      const goodFeedback = [];

      function cardUpdate(value, index, array){
      
          txt += "tr>td>" + value.name + "/td>" + "td>"  + value.feedback + "/td>" + "td>" + value.rating + "/td>" + "td>" + value.product + "/td>/tr>";
          }

      function reset_form() {
          document.getElementById("name").value = "";
          document.getElementById("feedback").value = "";
          document.getElementById("rating").value = "";
          document.getElementById("product").value = "";
      }

      function addFeedback(){
  
          var nametmp, feedbacktmp, ratingtmp, producttmp, celltmp

          nametmp = document.getElementById("name").value;
          feedbacktmp = document.getElementById("feedback").value;
          ratingtmp = document.getElementById("rating").value;
          producttmp = document.getElementById("product").value;

          celltmp = {

              "feedback": feedbacktmp,
              "rating": ratingtmp,

      if ((ratingtmp =="Excellent") | (ratingtmp == "Good")){
      
          goodFeedback.unshift(celltmp);
          document.getElementById("positive").innerHTML = "";
          txt = "table id='table_feedback'>";
          txt += 'thead> th id="first_row"> Name /th>th id="second_row"> Feedback  /th>th id="third_row"> Rating  /th>th id="fourth_row"> Product /th>/thead>';
          goodFeedback.forEach(cardUpdate);
          txt += "/table>";
          document.getElementById("positive").innerHTML = txt;

      else if ((ratingtmp =="Poor") | (ratingtmp == "Disappointing")) {
      
          badFeedback.unshift(celltmp);
          document.getElementById("negative").innerHTML = "";
          txt = "table id='table_feedback'>";
          txt += 'thead> th id="first_row"> Name /th>th id="second_row"> Feedback /th>th id="third_row"> Rating /th>th id="fourth_row"> Product /th>/thead>';
          badFeedback.forEach(cardUpdate);
          txt += "/table>";
          document.getElementById("negative").innerHTML = txt;
      }   
      
      reset_form();

      }

Express NodeJS (New Date and If Function)

My express NodeJS repose form was made like what we did in the practical however I added two new functions. One is a new date function and one is an IF function. I researched the date function on various forums including stackoverflow.

First I made a new function called addDays you can see the code below

    
      function addDays(date, days) {
        var result = new Date(date);
        result.setDate(result.getDate() + days);
        return result;
        }

This code helps make the delivery time variable

    
      let orderdate = new Date();
      deliverytime = addDays(orderdate, 7);

      deliverytime.toLocaleDateString()

You can then set the delivery time to be however many days more than the orderdate in this case it is 7. This function is very nice for an Express Order Form like what we have done for this website. You can then display the expected deliverytime when someone orders a product using "toLocaleDateString()"

I also created a new IF function. The IF function was easy to implement and I didn't really need to research this technique the course material and my own intution was enough to create this function. This function was used to apply a different postage fee depending on what state you order from. You can see the code below

    
      function costState(state) {
        if (state == "VIC") {
            return 5.00;
        } else {
            return 10.00; }       
    };

For my website the store is located in VIC so obviously the postage would be cheaper there. For every other state it is 10 dollars. You could add more else if statements to have different postage fees for all of the different states.
